Input Output dan Sistem Bus

by Banny Juanda - 1201226080 - TELKOM UNIVERSITY JAKARTA

Pengatar

Input/Output (I/O) dan Sistem Bus merupakan dua konsep dasar dalam komputasi moder. I/O mengacu pada pengiriman dan penerimaan data antara komputer dan perangkat peripheral seperti keyboard, mouse, monitor dan printer. Di sisi lain, Sistem Bus adalah jalur komunikasi yang menghubungkan berbagai komponen perangkat keras dari sistem komputer. Kedua konsep ini memainkan peran penting dalam memungkinkan interaksi manusia-komputer dan memfasilitasi komunikasi dan koordinasi antara berbagai komponen perangkat keras dari sistem komputer.

Input dan Output (I/O)

Mekanisme pengiriman data melalui aliran data yang diproses dari perangkat input ke perangkat output. Fungsi dari I/O adalah sarana komunikasi antara pengguna dengan komputer.

Perangkat Peripheral (I/O)

Peripheral adalah perangkat eksternal yang tersambung dalam sistem CPU melalui modul I/O melalui line kontrol, status, dan data.

Beberapa kategori perangkat peripheral :

  • Human Readable Machine : Perangkat fisik yang digunakan sebagai alat komunikasi pengguna komputer.
    Contoh : Monitor, keyboard, mouse, printer, joystick, diskdrive.
  • Machine Readable Machine : Perangkat fisik yang berhubungan dengan peralatan atau hanya bisa terbaca oleh komputer.
    Contoh : Modul sensor dan tranduser untuk monitoring dan kontrol suatu peralatan atau system, piringan disk.
  • Communication : Perangkat yang digunakan untuk komunikasi jarak jauh.
    Contoh : NIC, modem.

Input / Ouput Port

Tempat konektor Input Output melalui control processor, berfungsi sebagai jalur masukan perintah untuk komputer dan jalur keluar dari hasil perintah komputer.

Konfigurasi Input / Ouput

INPR dan OUTR Register

Tempat peyimpanan data sementara dalam CPU selama proses eksekusi. Apabila terjadi proses eksekusi data dalam register dikirim ke ALU untuk diproses, hasil dari eksekusi nanti diletakan ke register kembail.

AC / Unit Control

Unit kontrol akan menghasilkan sinyal yang akan mengontrol operasi ALU dan perpindahan data ke dan dari ALU.

FGI / FGO Flags

Flag diset ALU sebagai hasil dari suatu operasi, misalnya: overflow flag diset 1 bila hasil komputasi melampaui panjang register tempat flag disimpan.

Proses Input / Ouput

  • CPU memeriksa status I/O modul device
  • I/O modul mengembalikan status
  • Jika sudah siap, CPU melakukan permintaan pengiriman data
  • I/O modul mendapatkan data dari device
  • I/O modul melakukan pengiriman data ke CPU
  • Output

Teknik Operasi Input / Ouput

1. Programmed I/O

Pada programmed I/O, data saling dipertukarkan antara CPU dan modul I/O. Kelemahan dari programmed I/O adalah membuang waktu CPU karena kecepatan pengiriman data pada CPU lebih cepat daripada perangkat peripheral, saat CPU menerima perintah lain, CPU harus menunggu sampai proses pengiriman data selesai.

Proses :

  • CPU melakukan permintaan operasi perintah kepada I/O
  • CPU mempunyai kontrol pada Sensing Status, perintah read/write, dan pengiriman data I/O
  • CPU memberi alamat, menentukan modul I/O dan perangkat peripheral
  • Modul I/O menjalankan operasi
  • Modul I/O memberikan status bits
  • Modul I/O tidak dapat melakukan interupsi kepada CPU
  • CPU memeriksa status bits secara periodik

2. Interrupt Driven

CPU mengeluarkan perintah pada modul I/O, disaat yang sama CPU akan melakukan perintah lainnya. Setelah modul I/O selesai menjalankan instruksi, modul I/O akan melakukan interupsi pada CPU bahwa tugasnya sudah selesai. CPU melakukan multitaskin beberapa perintah sekaligus, sehingga tidak ada waktu tunggu bagi CPU saat mengirim data.

Proses :

  • I/O controller menerima perintah, misal read.
  • I/O controller melaksanakan perintah pembacaan dari peripheral dan meletakan paket data ke register data I/O controller, sementara CPU mengerjakan yang lain.
  • I/O controller mengeluarkan sinyal interupsi ke CPU melalui saluran kontrol.
  • CPU meminta data.
  • Modul meletakkan data pada bus data dan modul siap menerima perintah selanjutnya.

3. Direct Memory Access (DMA)

CPU menyerahkan kerja I/O kepada Direct Memory Access. CPU hanya terlibat pada awal (memberi perintah pada DMA) dan akhir proses. Hanya melibatkan CPU dan DMA, sehingga kinerja CPU tidak akan banyak terganggu dengan interupsi.

Proses :

  • CPU memberikan perintah ke modul DMA (read/write, alamat perangkat I/O, lokasi awal memori, jumlah word).
  • CPU melanjutkan pekerjaan lain.
  • Modul DMA memindahkan seluruh blok data secara langsung ke memori tanpa harus melalui CPU.
  • Modul DMA mengirimkan signal interrupt ke CPU.
  • CPU dapat menjalankan proses lainnya.

Sistem Bus

Sistem Bus dapat dianggap sebagai sistem transportasi internal komputer. Sistem Bus membawa alamat, data, dan informasi kontrol.

Interkoneksi Bus

Bus merupakan lintasan komunikasi yang menghubungkan dua atau lebih komponen komputer. Diperlukan aturan main agar tidak terjadi kerusakan data yang ditansmisikan, walaupun digunakan bersama namun dalam satu waktu hanya ada sebuah perangkat yang dapat menggunakan bus.

Lebar Bus

Semakin lebar bus maka semakin besar data yang dapat dikirim salam satu waktu.
Semakin besar bus alamat, maka semakin banyak memori yang dapat diakses.

Bagian Saluran Sistem Bus

Saluran Data

Saluran Data adalah jalur bagi perpindahan data antar modul, secara kolektif jalur ini disebut bus data. Pada Umumnya jumlah saluran terkait dengan panjang word, misalnya 8, 16, 32 saluran.
Tujuan dari Saluran Data adalah mengirim word dalam satu waktu. Lebar bus dapat dikatakan sebagai jumlah saluran dalam bus data dengan satuan bit, misalnya lebar bus 16 bit.

Saluran Alamat

Saluran Alamat berfungsi untuk menspesifikasi sumber dan tujuan data pada bus data, mengirim alamat word pada memori yang akan diakses CPU, perangkat modul komputer saat CPU mengakses suatu modul.
Semua peralatan yang terhubung dengan sistem komputer harus memiliki alamat supaya dapat diakses. Contohnya mengakses port I/O, maka port I/O harus memiliki alamat hardwarenya.

Saluran Kontrol

Saluran Kontrol berfungsi untuk mengatur bus data, bus alamat dan seluruh modul yang ada. Karena bus data dan bus alamat digunakan oleh semua komponen maka diperlukan suatu mekanisme kerja yang diatur melalui bus kontrol ini.
Sinyal-sinyal kontrol terdiri atas

  • Sinyal pewaktuan
  • Sinyal-sinyal perintah

Jenis-jenis Bus

Dedicated Bus

Dedicated Bus adalah bus yang dibedakan menjadi bus yang khusus menyalurkan data tertentu, misalnya paket data saja, atau alamat saja.

Multiplexed Bus

Multiplexed bus adalah bus yang dilalui oleh informasi yang berbeda baik data, alamat maupun sinyal kontrol.
Keuntungan dari Multiplexed bus adalah hanya memerlukan saluran yang sedikit sehingga dapat menghemat tempat.
Kerugian dari Multiplexed bus adalah kecepatan mengirim data menurun dan memerlukan mekanisme yang kompleks untuk mengurai data yang telah dimultiplex.

Contoh Bus

  • Industry Standar Architecture (ISA)
  • Peripheral Component Interconnection (PCI)
  • Universal Standard Bus (USB)
  • Small Computer System Interface (SCSI)
  • Bus P1394 / Fire wire

Metode Arbitrasi

Pada metode tersentral diperlukan pengatur bus sentral atau arbiter yang bertugas mengatur penggunaan bus oleh modul. Arbiter bisa suatu modul atau bagian fungsi CPU.

Pada metode terdistribusi, setiap modul memiliki logika pengatur akses yang berfungsi sebagai mengatur pertukaran data melalui bus.

Kedua metode arbitrasi pada intinya menugaskan suatu perangkat bisa modul I/O ataupun CPU bertindak sebagai master kontrol pertukaran.

Timing-Sinkron

Sebuah transmisi 1-0 disebut siklus waktu dan menentukan besarnya slot waktu. Metode pewaktuan sinkron terjadinya event pada bus yang ditentukan oleh sebuah pewaktu (clock). Semua perangkat modul pada bus dapat membaca atau mengetahui siklus clock. Biasanya satu siklus untuk satu event.

Model ini mudah diimplementasikan dan cepat namun kurang fleksibel menangani peralatan yang beda kecepatan operasinya. Biasanya digunakan untuk modul-modul tertentu yang sudah jelas karakteristiknya.

Timing-Asinkron

Kecepatan kerja modul yang tidak bersamaan. Event yang terjadi pada bus tergantung event sebelumnya sehingga diperlukan sinyal-sinyal validasi untuk mengidentifikasikan data yang dikirim.

Sistem ini dapat menggabungkan kerja modul-modul yang berbeda kecepatan maupun teknologinya, asalkan aturan kirimnya sama.

Transfer Data

Data yang tersimpan dalam memori dapat diakses dan dieksekusi CPU melalui perantara bus. Melihat hasil eksekusi melalui monitor juga menggunakan sistem bus. Kecepatan komponen penyusun komputer harus diimbangi kecepatan dan manajemen bus yang baik.
